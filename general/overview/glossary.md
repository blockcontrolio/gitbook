# Glossary

General terms used throughout this integration guide.

---

| Term | Definition |
|---|---|
| **BlockControl** | The technical processing center and white-label stablecoin infrastructure platform. Referred to as "the platform" or "BC" throughout this guide. |
| **Arbitrum** | The public Ethereum Layer 2 blockchain network used by BlockControl for on-chain settlement of all EMT transactions. |
| **Arbiscan** | The public blockchain explorer for Arbitrum (`arbiscan.io`). Used for independent transaction verification and reconciliation. |
| **EMT** | Electronic Money Token. A MiCAR-compliant crypto-asset referencing a single fiat currency (e.g., EUR), issued by a licensed EMI. |
| **Counterparty** | A registered institution (bank or EMI) in the BlockControl platform. Each counterparty has its own accounts, tokens, and users. |
| **Counterparty GUID** | A unique identifier assigned to each counterparty during onboarding. Included in all API request URLs. |
| **Account** | An on-chain smart wallet belonging to a counterparty. Accounts have types (Admin, Issuer, Operator, Distributor, Client) that determine their capabilities. |
| **Account GUID** | A unique identifier for an account within a counterparty. Used as `sender_id` and `receiver_id` in payment requests. |
| **Emission Account** | An account of type **Issuer** — responsible for minting and burning EMT supply. Equivalent to the emission entity in the processing flow. |
| **Processing Account** | An account of type **Distributor** — distributes tokens between issuer and client accounts, and across counterparties. |
| **Customer Account** | An account of type **Client** — holds end-user balances and performs transfers. |
| **Employee** | A platform user (human operator) belonging to a counterparty. Employees have roles: Admin, User, or Read-Only. |
| **Smart Account** | An ERC-4337 account abstraction wallet deployed on Arbitrum. Enables gas sponsorship via Paymaster and on-chain role enforcement. |
| **Paymaster** | A smart contract that sponsors gas fees for operations performed by smart accounts, removing the need for accounts to hold ETH. |
| **MPC** | Multi-Party Computation. A cryptographic technique used by BlockControl's WaaS provider (Dfns) to secure private key management without any single point of compromise. |
| **WaaS** | Wallet-as-a-Service. BlockControl integrates with **Dfns** as its WaaS provider for MPC-based transaction signing. |
| **JWT** | JSON Web Token. A signed token used for API authentication. Can be generated by the bank or issued by BlockControl's auth server. |
| **Session Token** | A time-limited JWT issued after successful login. Must be included in the `Authorization` header of all API requests. |
| **GUID** | Globally Unique Identifier. A UUID-format string used as the primary identifier for all entities (counterparties, accounts, payments, etc.). |
| **Asset Code** | The ticker symbol of an EMT token (e.g., `FEUR`). Used in payment requests to specify which token to transfer. |
| **Asset Issuer** | The on-chain contract address of the EMT token on Arbitrum. Together with `asset_code`, uniquely identifies a token. |
| **Transfer Hash** | The Arbitrum transaction hash produced when an on-chain operation is confirmed. Can be looked up on Arbiscan. |
| **Multisig** | A four-eyes approval mechanism requiring two independent parties to co-sign a transaction before it is broadcast to Arbitrum. |
| **RBAC** | Role-Based Access Control. The permission model governing what platform users and accounts can do, enforced at both API and smart contract level. |
| **MiCAR** | Markets in Crypto-Assets Regulation. The EU regulatory framework governing EMT issuance and operation, which BlockControl is designed to comply with. |
| **Webhook** | An HTTP callback registered by the bank that BlockControl calls when a transaction status changes or an invoice is completed. |
| **Inter-Counterparty Transfer** | A transfer of tokens from one counterparty's processing account to another counterparty's processing account. Also called an inter-bank transfer. |
