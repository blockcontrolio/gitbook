# Authentication

How to log in to the system with employee credentials.

---

In order to work with the BlockControl platform, you must authenticate and obtain a session token. This token must be included in every subsequent API request.

{% hint style="success" %}
If you use [your own JWT](bank-authorization.md) generated by the bank's identity provider, you do not need to perform authentication via the BlockControl API.
{% endhint %}

The purpose of authentication is to restrict access to crypto-identities, account information, and financial operations only to those who are explicitly authorized.

Please use the following host for all authentication requests:

```
https://api.{your_alias}-stage.blockcontrol.io   ← Stage
https://api.{your_alias}.blockcontrol.io          ← Production
```

---

## What steps should you do for authentication?

**Precondition**: The employee must already be registered. See [Create employee](../employee/create-employee.md).

### Step 1 — Log in

```http
POST /api/v1/auth/login
Content-Type: application/json

{
  "email": "operator@mybank.com",
  "password": "your_secure_password"
}
```

**Response:**

```json
{
  "session_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 1200
}
```

Store the `session_token` value.

---

### Step 2 — Use the token in all requests

Include the session token in the `Authorization` header for every API call:

```
Authorization: Bearer ${session_token}
```

**Example:**

```http
GET /api/v1/counterparty/{counterparty_guid}/accounts
Authorization: Bearer eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...
```

{% hint style="warning" %}
The session token is valid for **1200 seconds** (20 minutes) by default. After expiry, repeat the login step to obtain a new token. The bank may configure the validity period during onboarding.
{% endhint %}

---

### Step 3 — Refresh the token (optional)

To extend a session without re-entering credentials:

```http
POST /api/v1/auth/refresh
Authorization: Bearer ${session_token}
```

**Response:**

```json
{
  "session_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires_in": 1200
}
```
